Best Features:

Feature #1
Today, “gymfluencers” and health and fitness are a big part of social media culture. 
However, one of the biggest issues is that people struggle to follow widespread workouts 
plans and splits. Everyone reacts to the same health plan differently, and there is no
 one-size-fits-all solutions for everyone's fitness goals.

The main goal of our app is to create a tailored and comprehensive fitness plan 
for the user. It begins with setting an initial fitness goal, whether that is gaining
 muscle, losing weight, or any kind of fitness endeavor. Using machine learning algorithms 
 that iterate and learn the users habits and preferences, the app delivers a user-specific 
 regimen that allows the user to achieve their goals in the most effective way. 

feature #2
The system features a comprehensive system for generating tailored workout plans for the user. 
First it begins with an initial workout plan based on the context the user gives (fitness goal, 
injuries, preferences, etc). Then....

The user receives the workout plan
The user logs what they actually did, what they did and did not find useful
The Machine Learning algorithm reads this information and iterates on it for the next workout plan
Then the steps are repeated every week, generating a unique and custom workout 

feature #3
Additionally, our system compares the user's dietary history against their goals and calculates what nutrients they are still missing. Every candidate meal is then scored across three factors: how much it closes the gap, how well it fits their long-term goals, and whether it includes their preferred foods (based on their dietary history). Before anything is shown, every meal is also filtered against the user's allergy list. If the user is not happy with the suggestions our system provides, they can submit feedback to our AI, which will regenerate the plan and run another safety check before saving it. As a result, the user will get a personalized, ranked, safety-validated meal plan that continuously adapts. 



Major Challenges and Solutions

Some of our key challenges and solutions were
Identifying functional requirements and use cases that would be useful for the project
Making sure that our UML diagrams and Database diagrams matched
Identifying what we wanted the app to look like and how we wanted users to interact with the app (minimalism vs intricate ui)


Functional Requirements

Our requirements started off as vague and overlapped user features as well as also focusing on back end calls, such as “The system uses reliable API calls to the database”
In order to fix these issues we broke down our requirements into single features, focused more on what the user sees and how they interact with the application as well as rewording requirements.

Making sure that our UML, ER, and Database diagrams matched:
Problem: We had seperate teams working on the three different types of diagrams. This meant that it was harder 
to have a form of consistency when it came to attributes, relationships, and entites. Our app also has many parts
to it that are important. Many parts meant that the models would be larger and have more relationships. Making sure that
each model had the same relationship was important and hard for us because it the relationships define our app, and each 
person may have differing interpatations to it. This basically all boils down to CONSISTENCY


UI Diagrams

The UI diagram was a little bit difficult to create due to it being the first time we drafted what our application would look like. It lacked flow and consistency when first mocking up designs as well as stale and bland in some parts. It took us a while to finally visualize what a user might want when they first open up the tracker. Using inspiration from other health trackers made it easier to see how important information should be displayed. We also tried to simplify features in order to get the point across without overwhelming the designs and limited space
